## 背景
### 法政大学のバスの課題
法政大学は西八王子，めじろ台，相原駅（以下，3駅と呼ぶ）の3つが最寄り駅とされているがいずれも大学から徒歩圏内ではなく，通学にはバスが必須である。そこでCODE MATESはバス乗り換えアプリ「たまっぷβ版」を開発する運びとなった。たまっぷは3駅から法政大学間のバスの時刻を表示するアプリであり，3駅から法政大学・法政大学から3駅のバスの時刻がリアルタイムで分かるようになっている。
### 飲食店との連携
#### たまらぼ
2024年度にCODE MATESはSIC主催ビジネスコンテスト「チェンジメーカーズラボ in 多摩」（たまらぼ）に出場することになった。そこで，たまっぷにビジネスの要素を組み込む必要があった。そこで，たまっぷが3駅と法政大学という限定的な情報を提供していることに着目し，近隣の地域と連携することによって，地域活性化を図るといった「たまらぼ」の趣旨に即したテーマを決めた。また，法政大学多摩キャンパスの学食が値段が高く不便であることにも着目し，近隣の飲食店と連携していた。
#### 連携内容
当初は，
- めじろ台「藤」
- 西八王子「吾衛門」
- 橋本「コクテル堂」
- 八王子「ハイチーズ」

と連携していた。契約の内容としては，たまっぷにクーポンを掲載し，法政大学の学生であれば割引を受けることができるというものだ。掲載料は頂かず，飲食店はクーポンの掲載による広告効果があり，CODE MATESは学生にとって便利なクーポンが利用できるというメリットがあった。2026年4月現時点では，いずれの契約も期間が終了しており，現在はたまっぷ上でクーポンは掲示しておらず，当然店舗も割引の義務はない。

### 技術スタック
たまっぷにReactを導入する前では，PythonのWebフレームワークFlaskを用いていた。Flaskのテンプレートエンジンでサーバーサイドで時刻を計算して，適切なHTMLをサーバーサイドで生成しクライアントに返すという流れだった。
### 課題
#### 表示速度
当時は時刻の計算プロセスの一部にサーバーのレスポンスを待つ必要があった。LolipopサーバーでCGIスクリプトを用いてFlaskを動かしていたため，サーバーのレスポンスが遅くなり，最終的に時刻が表示されるまで時間がかかっていた。
#### CSSの煩雑化
基本的にフロントエンドのスタイリングはHTMLに素のCSSを記述していく形式であり，UIの構造が複雑になるにつれて`!important`の仕様や意図の不明瞭なCSSが増大し，メンテナンス性が著しく低下していた。
#### Webフレームワークの利用意義
当時は3駅と往復の計6パターンについてFlaskのレンダリング機能を用いていた。駅や往復をページ上で切り替える際にはページを読み込みなおす形となりUXが低いことが課題だった。クライアント側でスムーズな表示切替を実現するうえで，時刻の計算ロジックをサーバーサイドからクライアントサイドに寄せる必要が出てきた。たまらぼ出場時点ではFlaskは時刻表のJSONファイルを返すのみとなっており，ほとんどFlaskのWebフレームワークとしての利点を生かしていなかった。
### 移行
上記課題を踏まえて，Webフレームワークで時刻を計算するのではなく，静的ファイルを配信し全ての計算ロジックをクライアント側に寄せれば劇的に表示速度が改善するのではないかと当時の冨永は考察した。そこで，React＋tailwindcssといった全く新しい技術スタックでたまっぷを作り直した。React化以前から存在した時刻の計算ロジックはほとんど引き継いでいるので，時刻の計算については苦戦することはなかった。
## 概要
### jamstack
上記で述べた通り現在のたマップは静的ファイルの配信によって実現している。これはサーバーサイドで動く言語がないのでメンテナンスが簡単である上に，非常に多くのアクセスにも耐えうる構成である。基本的にファイルを配信する形なので，ほかの環境への移植がしやすく，cloudflare Pagesを用いれば，追加の設定をほとんどせずともCDNを生かした高速なファイル配信を実現できる。
### Vite + React
現たまっぷ・このプロジェクトではViteのReactのテンプレートを使用している。Viteのツリーシェイキング機能により高速で開発サーバーが起動することができる。また`npm run build`でプロダクションビルドを出力する際にはWebpackなどと同様にバンドルが利用できるうえ，Webpackよりはるかに高速にビルドをすることができる。
### 技術スタック
#### React
JavaScriptフレームワークのReactを採用することによって，DOM操作を劇的に改善している。また，エコシステムを生かして既存のライブラリを生かした開発も実現している。
#### tailwindcss
tailwindcssはユーティリティクラスと呼ばれるクラスをHTMLに直接羅列してスタイリングをするフレームワークである。
#### shadcn UI
shadcn UIは再利用可能なUIのコレクションである。用意されたUIの一部を改変して利用することができする。
#### GSAP
GSAPはアニメーションを作るためのライブラリである。
